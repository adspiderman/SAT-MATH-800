# Lesson 0.2 — How to Use This Course

This course simulates what a paid prep course does — structured content, deliberate practice, spaced review, and full simulation — but it only works if you follow the system. Here is the system.

## The learning loop (every lesson)

Every lesson in Units 1–4 follows the same structure. Use it the same way every time:

1. **Concepts** — read actively. Work each worked example on paper *before* reading its solution. If you got it right, skim the solution; if not, study it.
2. **Desmos angle** — most lessons show how to solve or verify with the calculator. Practice it immediately in the [testing calculator](https://www.desmos.com/testing/cb-digital-sat/graphing).
3. **Traps** — the specific ways the SAT will bait you on this topic. Read these twice.
4. **Practice set** — do all problems, no calculator help unless the problem says so, *then* check the answer key.
5. **Log misses** — every wrong answer goes in your [error log](../05-strategies/06-error-log.md) with the *reason* you missed it.

## The three phases of preparation

### Phase 1 — Content (Units 1–4)
Goal: **untimed accuracy.** You should be able to get ~95% of lesson problems right with no time limit before you worry about speed. Speed comes from fluency, and fluency comes from correct reps.

### Phase 2 — Strategy & mixed practice (Unit 5 + unit quizzes)
Goal: **choosing the fastest correct method.** Many SAT problems have a 3-minute "school way" and a 40-second "test way" (backsolving, plugging in numbers, Desmos). Phase 2 is where you build the instinct for which to use.

### Phase 3 — Simulation (Unit 6 + Bluebook tests)
Goal: **performance under real conditions.** Timed modules, no music, no pausing, answer everything. After each test, spend *at least as long reviewing as you spent testing*. An unreviewed practice test is worth almost nothing.

## Rules that separate 800-scorers from 700-scorers

1. **Never read a solution before attempting the problem.** Recognition is not recall.
2. **Review every miss until you can re-derive the solution cold**, then re-do it 3 days later from your error log.
3. **Classify every error**: concept gap, misread, arithmetic slip, or time pressure. Each has a different fix (see error log lesson).
4. **Don't grind only hard problems.** At 750+, points are lost on easy/medium questions via carelessness far more often than on hard ones via difficulty.
5. **One timed module beats three untimed ones** once you're in Phase 3.

## Time budget

- Each lesson: 45–90 minutes (concepts + practice).
- Unit quizzes: 25–35 minutes timed.
- Full practice module: 35 minutes + 45–60 minutes of review.
- Weekly total: see your [study plan](03-study-plans.md).

## What you need

- This repo, a notebook (or tablet) for working problems on paper — SAT scratch work is on paper on test day too.
- The [Desmos testing calculator](https://www.desmos.com/testing/cb-digital-sat/graphing) in a browser tab.
- Bluebook installed, with the official practice tests.
- Your error log (spreadsheet or the [template](../05-strategies/06-error-log.md)).
